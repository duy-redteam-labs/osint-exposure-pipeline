# Scope Definition

## Project
Professional-Style Exposure Assessment for OWASP Juice Shop in a controlled internal lab.

## Target
- OWASP Juice Shop instance deployed in the private lab
- Only lab-owned systems and services directly related to this target

## Purpose
The purpose of this project is to perform a safe and structured exposure assessment focused on reconnaissance, observation, evidence collection, normalization, scoring, and reporting.

This project is for training and portfolio development only.

## Allowed Activities
- Passive observation of the application in the browser
- Normal HTTP requests to public pages
- HTTP response header inspection
- Review of visible routes, static assets, and client-side artifacts
- Technology fingerprinting from publicly observable responses
- Documentation of metadata, route exposure, and technical clues
- Local processing, normalization, scoring, and reporting of collected evidence
- automated collection of public responses and client-side artifacts within the lab scope

## Prohibited Activities
- Exploitation of vulnerabilities
- Brute force attempts
- Credential attacks
- Unauthorized access attempts
- Privilege escalation
- Destructive testing
- Denial-of-service style activity
- Any action outside the internal lab environment
- Any state-changing action unless strictly necessary for harmless observation

## Expected Outputs
- Raw evidence files
- Normalized findings
- Risk-scored findings
- Exposure report
- Supporting screenshots and documentation

## Boundaries
All activity must remain inside the authorized internal lab. No external or real-world targets are included in scope.
